# Framework Outputs Directory

This directory contains all outputs generated by the Multi-Agent Test Automation Framework, organized by module and type.

## Directory Structure

```
outputs/
├── requirements/           # Story analysis requirements from Story Analyst Agent
│   ├── authentication/    # Authentication module requirements
│   ├── account-management/ # Account management module requirements
│   ├── transactions/       # Transaction module requirements
│   ├── bill-pay/          # Bill payment module requirements
│   └── general/           # General/unclassified requirements
├── test-cases/            # Generated test cases from Test Generator Agent
│   ├── authentication/    # Authentication module test cases
│   ├── account-management/ # Account management module test cases
│   ├── transactions/       # Transaction module test cases
│   ├── bill-pay/          # Bill payment module test cases
│   └── general/           # General/unclassified test cases
└── test-results/          # Test execution results from Test Executor Agent
    ├── authentication/    # Authentication module test results
    ├── account-management/ # Account management module test results
    ├── transactions/       # Transaction module test results
    ├── bill-pay/          # Bill payment module test results
    └── general/           # General/unclassified test results
```

## File Naming Conventions

### Requirements Files
- **Format**: `{story-id}-requirements-{timestamp}.md`
- **Example**: `AUTH-001-requirements-2025-10-05T19-30-00-000Z.md`
- **Content**: Story analysis output from Story Analyst Agent

### Test Case Files
- **Format**: `{story-id}-tests.md`
- **Example**: `AUTH-001-tests.md`
- **Content**: Generated test cases from Test Generator Agent

### Test Result Files
- **Format**: `{story-id}-results-{timestamp}.json`
- **Example**: `AUTH-001-results-2025-10-05T19-35-00-000Z.json`
- **Content**: Test execution results from Test Executor Agent

## Module Organization

Files are automatically organized by module based on the user story source:

- **authentication/**: Login, logout, MFA, social authentication
- **account-management/**: Registration, profile management, account settings
- **transactions/**: Money transfers, transaction history, deposits
- **bill-pay/**: Bill payment setup, payee management, scheduling
- **general/**: Stories without specific module classification

## Benefits of This Structure

✅ **Organized**: Clear separation by module and output type  
✅ **Consistent**: Uniform structure across all output types  
✅ **Scalable**: Easy to add new modules and story types  
✅ **Clean**: Root directory remains uncluttered  
✅ **Traceable**: Clear path from story → requirements → test cases → results  

## Migration

This structure replaces the previous layout:
- `generated-prompts/requirements/` → `outputs/requirements/{module}/`
- `generated-test-cases/` → `outputs/test-cases/`
- Root-level JSON files → `outputs/test-results/{module}/`
